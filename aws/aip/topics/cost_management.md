# コスト管理（Cost Management）

## Cost Explorer + タグ戦略（コスト配分追跡）

どのビジネス機能がBedrockコストを消費しているかをタグで可視化する。

```
# Bedrock呼び出し時にタグを付与
bedrock:feature = "summarization"
bedrock:team    = "customer-support"
bedrock:env     = "production"

→ Cost Explorerでタグ別フィルタ
→「要約機能が月$500、サポートが月$300」と把握
→ コスト超過の機能を特定してモデルを見直す
```

（マルチテナント文脈では Cost Explorer / 請求側でテナント別のコストを分解できる）

## AWS Budgets（コスト予算・アラート）

Cost Explorer が「事後分析」なのに対し、Budgets は**予算上限の設定とアラート**が目的。

```
設定例：
  Bedrock の月次予算 = $3,000
  → 80%（$2,400）到達 → メール通知
  → 100%（$3,000）到達 → SNS → Lambda で自動対応（モデル切り替え等）
```

- ⚠️ **訂正**：AWS Budgetsに機械学習ベースの「コスト異常検出」機能はない（過去の記述の誤り）。Budgetsは**人が設定した固定閾値**と実コストを比較するだけで、過去の傾向を学習して動的にベースラインを構築する機能は持たない。機械学習によるコスト異常検知は別サービスの**AWS Cost Anomaly Detection**が担う（下記参照）
- コストデータは1日最大数回更新されるため、更新サイクルと通知が重なると急増の把握がさらに遅れる構造がある
- CloudWatch 異常検出（トークン数・レイテンシー等の**メトリクス**ベース）との使い分け：
  - トークン数・レイテンシー等のメトリクス異常 → CloudWatch 異常検出（推定請求額メトリクスも対象にできるが、単一集計値でサービス別の根本原因分析はできない）
  - **金額そのものの異常をMLでパターン検知** → **AWS Cost Anomaly Detection**

## AWS Cost Anomaly Detection（CAD）：機械学習によるコスト異常検知（公式確認済み）

AWS Budgetsとは別サービス。Cost Explorerの請求データを**機械学習モデルで継続的に評価**し、過去の傾向・季節性を学習した上で異常な支出パターンを自動検知する、コスト管理に特化したサービス。

```
Cost Explorerの請求データ処理 → 最大約24時間の遅延
  ↓
CADが1日約3回の頻度で評価を実行
  ↓
異常検知 → AWSサービス・アカウント・リージョン・使用タイプの
           4次元で根本原因候補を自動ランク付けして提示
```

### モニタータイプの違い（誤答しやすいポイント）

| | **AWSマネージドモニター**（`MonitorType: DIMENSIONAL`） | **カスタマーマネージドモニター**（`MonitorType: CUSTOM`） |
|---|---|---|
| 対応ディメンション | **AWSサービス**／リンクアカウント／コスト配分タグ／コストカテゴリの**いずれか1つを丸ごと自動追跡** | リンクアカウント／コスト配分タグ／コストカテゴリの中から**特定の値を最大10件まで個別指定** |
| 「AWSサービス」ディメンション | ✅ 対応（全サービスを自動監視、新規サービスも自動で追加設定不要で取り込まれる） | ❌ **非対応**（特定サービス単体を指定した「サービス横断型カスタムモニター」は作れない） |
| 向いている要件 | 「全AWSサービスを手動設定なしで自動監視」 | 「特定のリンクアカウント/タグ/カテゴリだけを個別監視」 |

→ 「Bedrockを含む全サービスを自動監視・運用負荷最小」が要件なら**AWSマネージドモニター＋AWSサービスディメンション**一択。カスタマーマネージドでBedrockを個別指定しようとする選択肢は、そもそもAWSサービスディメンションを扱えないため成立しない

### アラートサブスクリプションの種類

| 種類 | 通知タイミング |
|---|---|
| **個別アラート（individual alert）** | 異常検知の**都度、即時**にSNSトピックへ通知。最速で担当者に届けたい場合はこれ |
| 日次サマリー | 1日分をまとめてメール送信（到達が遅れる） |
| 週次サマリー | 1週間分をまとめてメール送信（さらに遅れる） |

- SNSトピック経由でSlack・Amazon Chime等への転送も可能
- 「できるだけ早く通知」という要件では**個別アラート**を選ぶ（日次/週次サマリーは不正解の定番）
