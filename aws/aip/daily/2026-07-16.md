- AWS Secrets Manager のネイティブ自動ローテーション（Parameter Store/DynamoDB/Lambda環境変数にはいずれも自動ローテーション機能がない）と、リソースベースポリシー＝AWSの暗黙デフォルト権限ではなく明示的に設定するポリシーだという誤解を訂正（KMSデフォルトキーポリシーが数少ない自動生成の例外） → [security_governance.md](../topics/security_governance.md)

- S3バケットポリシーによるCMK暗号化の強制（デフォルト暗号化だけでは`PutObject`側で上書き可能）とKMS対称型/非対称型キーの違い（対称型のみS3 SSE-KMS/OpenSearch保存時暗号化に対応）を、仕組みから追記 → [security_governance.md](../topics/security_governance.md)
- Amazon Lex V2の基礎（インテント・スロット・フルフィルメント・NLU）を完全初見から新規メモ。Assisted NLU（入力理解の精度向上）と応答生成は別レイヤーという区別、Bedrock AgentsとLex V2の役割の違いも整理 → [ai_services.md](../topics/ai_services.md)
- Strands Agents SDK（`@tool`デコレータでのツール定義、SDK側にパラメータ自動検証機構はない）とBedrock Agents（アクショングループ/OpenAPIスキーマ）の製品レイヤーの違い。Strands組み込みのマルチエージェント協調パターン（Workflow/Graph/Swarm）を新規追加。「コード最小限」＝配線ロジックを自作しないことであり、コード行数の多寡ではないという解釈の訂正 → [bedrock_agents.md](../topics/bedrock_agents.md)
- Bedrock AgentCore Evaluations（2026-03-31 GA、組み込み評価子`Builtin.GoalSuccessRate`/`Builtin.ToolSelectionAccuracy`、オンライン評価/オンデマンド評価の2モード）を新規追加。CreateEvaluationJob（ModelEvaluation/RagEvaluationのみ）との対象範囲の違いを整理 → [bedrock_agents.md](../topics/bedrock_agents.md)
- SageMaker Clarifyのバイアスドリフトモニタリング（本番エンドポイント向け、Bedrockとは非ネイティブ統合）を新規追加。既存の「Bedrock+Clarifyの制約」節と矛盾しない具体例として整理 → [model_evaluation.md](../topics/model_evaluation.md)
