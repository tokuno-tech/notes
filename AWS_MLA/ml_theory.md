# ML理論・手法・評価指標 まとめ

## データ前処理

### スパース（Sparse）
データの大部分が0や欠損値で、実際の値（非ゼロ）がごく少ないデータ。
例：テキストのBag of Words（100単語中10単語しか出現しない）。

### 対数変換（Log Transformation）
データの値を対数（log）に変換する前処理手法。
右裾が長い分布（年収、売上等）に有効。外れ値の影響を圧縮し、線形モデルで扱いやすくなる。
※ゼロが含まれる場合は `log(x+1)`（log1p）を使う。

### 右裾が長い分布（Right-skewed）
大多数が低い値に集中し、一部の高い値が右方向に点在する分布。
例：年収（大多数は低〜中収入、一部が超高収入）、SNSフォロワー数。

### 層化抽出
データの分布を維持したままサンプリングする手法。
学習データと評価データのバランスを保つ。

### スキーマ
データの構造・形式を定義するもの。各カラムの名前・データ型・制約を指定。
- Spark DataFrame：固定スキーマ（事前定義必要）
- AWS Glue DynamicFrame：柔軟スキーマ（後付け・自動推論）

### アノテーション
データに正解ラベルを付ける作業。教師あり学習のためのデータ作成。

### DataFrame
**行と列を持つ表形式（テーブル）データ構造**。データ分析・前処理の基本単位。
- 行＝レコード（サンプル）、列＝特徴量
- 各列ごとに異なるデータ型を持てる（数値・文字列・日時等）
- 主な実装：
  - **pandas DataFrame**：Pythonの単一マシン向け、メモリ内処理
  - **Spark DataFrame**：分散処理向け、固定スキーマ（事前定義必要）
  - **Glue DynamicFrame**：柔軟スキーマ（後付け可、ネストデータに強い）
  - **Dask DataFrame**：pandas互換の分散処理版
- 操作：列選択、フィルタ、集計、結合、ピボット、欠損値処理など
- MLA出題：Spark DataFrameとDynamicFrameの違い、pandasでの前処理コード読解で頻出

### ビン分解（Binning / 離散化 / バケット化）
**連続値を区切ってカテゴリ化**する前処理手法。
例：年齢 → 「20代/30代/40代/...」、年収 → 「低/中/高」

**メリット**
- 外れ値の影響を緩和
- 非線形な関係を線形モデルでも扱える
- 特徴量を解釈しやすくなる
- 過学習を抑制できる場合がある

**主な分け方**
- **等幅ビン（Equal-width）**：値の範囲を等間隔で分割（例：0-10, 10-20, ...）
- **等頻度ビン（Equal-frequency / 分位数）**：各ビンに同数のサンプルが入るよう分割
- **クラスタリングベース**：K-means等で自動グルーピング
- **ドメイン知識ベース**：業務的に意味のある区切り（年齢の世代など）

**デメリット**
- 情報損失（細かい違いが消える）
- ビン境界付近の挙動が不自然になることがある

**MLA出題：「年齢を世代別に分類して特徴量化」「外れ値の影響を抑える前処理」→ ビン分解**

## エンコーディング

### ラベルエンコーディング
カテゴリを連番（0, 1, 2...）に変換。
例：赤→0, 青→1, 緑→2
- 問題点：偽の順序関係が生じる
- 推奨用途：順序のあるカテゴリ、ターゲット変数

### 順序エンコーディング（Ordinal Encoding）
**順序のあるカテゴリ**を、その順位に応じた整数に変換する手法。
例：高卒=1, 学士=2, 修士=3, 博士=4 ／ 不満=1, 普通=2, 満足=3

**メリット**
- 1列で表現できる（次元増えない）
- カテゴリの順序関係を保持できる
- 線形モデル（ロジ回帰等）で「1段階上がるとlog-oddsが○○変化」と自然に解釈可

**注意点**
- **順序のないカテゴリ（部門・色等）に使うと偽の順序が生まれてNG** → ワンホットを使う
- 等間隔を仮定する点に注意（高卒→学士と学士→修士のジャンプが同じとモデルは見なす）

**ラベルエンコーディングとの違い**
- ラベルエンコーディング：単に連番を振るだけ（順序を意識しない）
- 順序エンコーディング：**意味的な順序を反映した整数**を振る
- 実装は似ているが、適用判断が違う（順序があるか／ないか）

#### 順序エンコーディング vs ターゲットエンコーディング
| | 順序エンコーディング | ターゲットエンコーディング |
|---|---|---|
| 値の決め方 | カテゴリの**順位**（1,2,3...） | カテゴリの**目的変数の平均**（0.35, 0.20...） |
| 目的変数を使う | ❌ 使わない | ✅ 使う |
| リーケージ | なし | **あり**（K-fold必須） |
| 適用条件 | 順序があるカテゴリ | 高カーディナリティ・順序問わず |
| 例（学歴で退職予測） | 高卒=1, 学士=2, 修士=3, 博士=4 | 高卒=0.35, 学士=0.20, 修士=0.15, 博士=0.10 |

→ **「順位」を埋めるか「目的との関係」を埋めるかの違い**。
→ 学歴のように順序が明確 → 順序エンコーディング
→ 郵便番号・ユーザIDのような大量カテゴリ → ターゲットエンコーディング

**MLA出題：「順序のあるカテゴリ（学歴・評価レベル）」→ 順序エンコーディング**

### バイナリエンコーディング
カテゴリを固定ビット数のバイナリコードで表現。
例：8カテゴリ → 3ビット（2^3=8）
- 高カテゴリ数（数千個）で次元削減に有効

### ワンホットエンコーディング
カテゴリを二値ベクトルで表現。
例：赤→[1,0,0], 青→[0,1,0]
- 名義カテゴリに安全だが、カテゴリ数↑で次元爆発

### ダミー変数展開（Dummy Variable Encoding）
ワンホットの**1列削除版**。N個のカテゴリをN-1列で表現する。
例：赤/青/緑 → 「青ダミー[0/1]」「緑ダミー[0/1]」（赤は両方0で表現＝基準カテゴリ）

**ワンホットとの違い**
| | ワンホット | ダミー変数展開 |
|---|---|---|
| 列数 | N列 | N-1列 |
| 多重共線性 | 発生する（合計が必ず1） | 回避できる |
| 用途 | 木系モデル全般 | **線形回帰・ロジスティック回帰**で必須 |

**ポイント**
- 線形モデルでは多重共線性で係数推定が不安定になる → ダミー変数展開が定石
- 木系（XGBoost等）はワンホットでも問題なし
- pandasの`get_dummies(drop_first=True)` がこれ

### ターゲットエンコーディング（Target Encoding / Mean Encoding）
カテゴリ値を、そのカテゴリにおける**目的変数の平均値（または確率）**で置き換える手法。
例（二値分類「成約率」）：東京→0.45, 大阪→0.30, 札幌→0.20

**メリット**
- 次元が増えない（1列）
- **高カーディナリティ**カテゴリで威力を発揮（郵便番号・ユーザID等）
- 目的変数との関係を直接特徴量に埋め込める → 精度向上しやすい
- 木系・線形モデル両方で有効

**デメリット・注意点**
- **データリーケージ**の危険大 → 学習データの目的変数を直接使うため
  - 対策：**Out-of-Fold（K-fold）**でエンコード値を計算する
  - 対策：**スムージング**（サンプル少ないカテゴリは全体平均に寄せる）
- カテゴリのサンプル数が少ないと過学習しやすい
- 学習/本番で目的変数分布がズレるとTrain-Serving Skewが発生

**他のエンコーディングとの使い分け**
| 手法 | 次元 | 高カーディナリティ | リーケージリスク |
|---|---|---|---|
| ワンホット | カテゴリ数分 | △（次元爆発） | なし |
| 頻度エンコーディング | 1列 | ✅ | 低 |
| ターゲットエンコーディング | 1列 | ✅ | **高（要対策）** |
| バイナリエンコーディング | log2(N)列 | ✅ | なし |

**MLA出題：「高カーディナリティで目的変数との関係を活かしたい」→ ターゲットエンコーディング（K-fold必須）**

### 頻度エンコーディング（Frequency / Count Encoding）
カテゴリ値を、そのカテゴリの出現回数（または出現割合）に置き換える手法。
例：東京→5000, 大阪→2000, 札幌→500

**メリット**
- 次元が増えない（1列のまま）→ 高カーディナリティでも次元爆発しない
- 計算が軽い、実装が簡単
- カテゴリの「人気度・希少度」を数値として表現できる（木系モデルと相性◎）
- ワンホットでスパースになりすぎる場合の代替

**使用しない方が良いパターン**
- 異なるカテゴリが**同じ頻度**を持つ場合、衝突して区別できなくなる
- 頻度とターゲットに相関がない場合、ノイズ特徴になる
- 学習データと本番データで**頻度分布が変わる**と精度劣化（Train-Serving Skew）
- 線形モデル：頻度の大小に偽の順序関係を生む可能性
- カテゴリ数が少なく均等な場合：ワンホットの方が情報を残せる

MLA出題：高カーディナリティ対策の選択肢としてバイナリ/ターゲットエンコーディングと比較される。

### 類似性エンコーディング（Similarity Encoding）
**表記ゆれや文字列カテゴリ**を、文字列の類似度に基づいてベクトル化する手法。
例：「Apple Inc.」「Apple inc」「APPLE INC」を**類似カテゴリとして近いベクトル**に変換
- 文字n-gram類似度（Jaccard / Levenshtein等）でカテゴリ間の距離を計算
- ワンホットでは別カテゴリ扱いになるが、類似性エンコーディングなら**表記ゆれを吸収**

**メリット**
- 表記ゆれ・スペルミス・大文字小文字違いに頑健
- 高カーディナリティな自由記述カテゴリ（会社名・地名・商品名等）に強い
- 完全一致クレンジングが困難なデータに有効

**デメリット**
- 計算コストが高い
- 距離指標の選び方で結果が変わる

**他エンコーディングとの比較**
| 手法 | 表記ゆれ対応 | 高カーディナリティ |
|---|---|---|
| ワンホット | ❌ | △ |
| ターゲット | ❌ | ✅ |
| 頻度 | ❌ | ✅ |
| **類似性** | **✅** | ✅ |

**MLA出題：「表記ゆれのある文字列カテゴリ」「自由記述の会社名/商品名のエンコード」→ 類似性エンコーディング**

### TF-IDF（Term Frequency - Inverse Document Frequency）
テキスト内で単語の重要度を数値化する手法。
- TF：文書内での単語の出現頻度
- IDF：全文書での希少性（逆数）
- TF-IDF = TF × IDF（文書内頻出かつ他文書で少ない単語を高評価）
- 出力：各文書を「単語ID → 重要度スコア」の**スパースベクトル**で表現
- 欠点：単語の意味的関係を考慮しない（同義語・文脈は捉えられない）

**仕組みのイメージ**
- 「the」「is」のようにどの文書にも出る単語：IDF低 → スコア低（雑音扱い）
- 特定文書だけに頻出する専門用語：TF高×IDF高 → スコア高（特徴的）

**使われるケース**
- **文書分類**（スパム判定、ニュースカテゴリ分類）
- **検索エンジン**でのクエリと文書の関連度スコアリング
- **キーワード抽出 / 要約**（スコア上位の単語を抜き出す）
- **類似文書検索**（TF-IDFベクトル間のコサイン類似度）
- **レコメンド**（コンテンツベース：商品説明文の類似度）
- ML前処理として**テキストを数値ベクトル化**（線形モデル/SVM/ナイーブベイズに投入）

**使わない方が良いケース**
- 文脈・意味理解が必要 → Word2Vec / BERT / Embedding 系へ
- 短文（ツイート等）でTFが伸びない → 工夫が必要
- 多言語混在で語彙爆発する場合

**MLA出題ポイント**
- 「テキストを軽量に数値化したい」「シンプルなテキスト分類」→ TF-IDF
- 「意味的類似性が必要」→ BlazingText / Embedding（TF-IDFは×）

## 学習・訓練

### ミニバッチサイズと勾配推定ノイズ
ミニバッチサイズは「1回の重み更新に使うサンプル数」のこと。
このサイズと**勾配推定のノイズ**には強い相関がある。

**仕組み**
- 勾配は本来「全データ平均」だが、計算コスト的に**ミニバッチで近似**する
- ミニバッチサイズが**大きい** → サンプル数多 → 勾配推定の**ノイズが小さく安定**
- ミニバッチサイズが**小さい** → ノイズが大きく**ジグザグ**に学習が進む

**サイズ別の特徴**
| サイズ | 勾配ノイズ | メモリ | 収束 | 汎化性能 |
|---|---|---|---|---|
| 小（8〜32） | 大（ノイズ多） | 少 | 不安定だが局所最適を抜けやすい | 良い傾向 |
| 中（64〜256） | 中 | 中 | バランス良い | 一般的に推奨 |
| 大（1024〜） | 小（安定） | 多 | 速いが鞍点に嵌まりやすい | 落ちる傾向（学習率調整必要） |

**ポイント**
- 大きいバッチで学習率も上げる「Linear Scaling Rule」が定石
- 大バッチ＋ウォームアップで安定化させるのが大規模学習の常套手段
- MLA出題：「バッチサイズを大きくしたら精度が落ちた」→ 学習率調整 or ウォームアップ導入

### ウォームアップスケジュールとエポック数の関係
**ウォームアップ（Warmup）** ＝学習率を**最初の数エポックだけ徐々に上げていく**スケジュール。

**目的**
- 学習初期は重みがランダム → 大きな学習率でいきなり更新すると発散しやすい
- 最初は小さい学習率で「ならし運転」、徐々に目標学習率まで上昇
- その後は通常のスケジュール（Cosine Decay / Step Decay 等）で減衰

**典型的なスケジュール**
```
学習率
  ↑
  │       ___________
  │      /           \____
  │     /                  \____
  │    /                        \
  │___/
  └──────────────────────────────→ エポック
   ウォームアップ   通常学習      減衰
   (例:5エポック)
```

**エポック数との関係**
- ウォームアップは通常**全エポック数の5〜10%**を目安に設定
- 総エポックが**10** → ウォームアップ **1エポック**程度
- 総エポックが**100** → ウォームアップ **5〜10エポック**程度
- 短すぎる → 安定化効果が薄い
- 長すぎる → 学習時間の無駄、本学習エポックが減る

**使い所**
- Transformer / BERT等の大規模モデル（事実上必須）
- 大バッチサイズ学習（Linear Scaling Ruleとセット）
- ResNet等のCNN学習でも有効

**MLA出題：「学習初期で損失が発散」「Transformer学習が不安定」→ ウォームアップ導入**

## 正則化・過学習対策

### L1正規化（Lasso）
ペナルティ = 係数の絶対値の合計。
不要な特徴の係数をピタッと0にする。特徴選択が必要な場合に有効。

### L2正規化（Ridge）
ペナルティ = 係数の二乗の合計。
全係数を均等に小さくする（0にはしない）。全特徴を幅広く使いたい場合に有効。

### Elastic Net
L1（Lasso）+ L2（Ridge）を組み合わせた正則化手法。
`Elastic Net = α × L1 + (1-α) × L2`
特徴量が多く、相関もある場合に有効。

### ドロップアウト（Dropout）
ニューラルネットワーク学習時に、ランダムにニューロンを一時的に無効化する手法。
- 学習中：指定確率（例：50%）でニューロンをOFF
- 推論時：全ニューロンを使用
目的：過学習を防ぐ、特定ニューロンへの依存を減らす。

### バッチ正規化（Batch Normalization）
各層への入力を正規化（平均0、分散1）する手法。
- 学習中：現バッチの統計量で正規化
- 推論時：学習中に記録した移動平均を使用
効果：学習の安定・高速化、大きな学習率が使える。

### 早期停止（Early Stopping）
検証ロスを監視して、改善が止まった時点で学習を終了する手法。
訓練ロスは減り続けるが検証ロスが増え始めたら停止 → 過学習を防ぐ。

## 不均衡データ対策

### SMOTE
少数クラスのサンプル間でk近傍法を使い、合成サンプルを生成する手法。
特徴量空間で新しいポイントを作るため、多様性のあるデータ拡張が可能。

### Balance Data変換（Data Wrangler）
Data Wranglerに組み込まれた不均衡処理機能。3つの手法から選択可能：
1. SMOTE：合成サンプル生成
2. ランダムオーバーサンプリング：少数派を複製
3. ランダムアンダーサンプリング：多数派を削減

### フォーカルロス（Focal Loss）
クラス不均衡問題で、難しいサンプルに重点を置く損失関数。
- 正解を外したサンプル：ペナルティを強く
- 正解を当てたサンプル：ペナルティを小さく
用途：物体検出、医療画像の異常検知。

## 次元削減

### 主成分分析（PCA）
高次元データの次元を圧縮しながら、なるべく情報を保持する手法。
データの分散が最大になる新しい軸（主成分）を順に見つけて圧縮。
- 教師なし学習（ラベル不要）
- 用途：計算コスト削減、可視化（2D/3D化）、ノイズ除去

## テキスト / NLP

### BlazingText
高速なテキスト処理アルゴリズム（SageMaker組み込み）。
単語埋め込みや分類を効率よく実行。

### BiLSTM-CRF
文脈理解（BiLSTM）とラベル整合性（CRF）を組み合わせたモデル。
主に固有表現抽出（NER）で使用。

### NERモデル（Named Entity Recognition）
テキストから固有表現（人名・地名・組織名・日付等）を自動識別・抽出するモデル。
- AWS実装：Amazon Comprehend（マネージドNER）
- カスタム：Comprehend Custom Entity Recognition

### NTM（Neural Topic Model）
大量ドキュメントのトピック分布を学習する教師なし学習アルゴリズム。
文書から自動的にトピック（話題）を発見する。
- LDAより高速・大規模対応
- 入力：単語数ベクトル
- 出力：各文書に対するトピックの重み（0〜1）
- トピック数は事前指定が必要

### Seq2Seq（Sequence-to-Sequence）
入力系列を別の系列に変換するエンコーダ・デコーダ型のニューラルネットワーク。
- エンコーダ：入力系列（単語列等）を固定長のベクトル（コンテキスト）に圧縮
- デコーダ：そのベクトルから出力系列を1トークンずつ生成
- 入出力の長さが異なってOK（可変長対応）
- 用途：機械翻訳、文章要約、音声→テキスト、チャットボット
- RNN/LSTM/GRUベースが基本。Attention機構で長文の精度改善
- SageMaker組み込みアルゴリズム「Seq2Seq」あり（教師あり、入出力ともトークン列）
- MLA出題：翻訳・要約タスクの選定問題で出る

### テンソル（Tensor）
多次元配列のこと。
- スカラー：0次元
- ベクトル：1次元
- 行列：2次元
- テンソル：3次元以上
例：画像 = 3次元テンソル（高さ × 幅 × チャネル）

## 機械学習アルゴリズム

### XGBoost（eXtreme Gradient Boosting）
**教師あり学習**の勾配ブースティング決定木（GBDT）アルゴリズム。分類・回帰の両方に対応。
- 弱い決定木を順番に積み重ね、前の木の誤差を次の木が補正していく
- **特徴量重要度（Feature Importance）** や SHAP値で「どの特徴が予測にどれだけ寄与したか」を可視化できる
- ⚠️ 寄与度＝相関・貢献度であり、**因果関係ではない**（因果推論は別分野）
- 欠損値の自動処理、正則化、並列学習が強み
- 構造化データ（テーブルデータ）の鉄板アルゴリズム
- SageMaker組み込みアルゴリズム（CSV / libsvm / Parquet対応）
- 主なハイパーパラメータ：`max_depth`, `eta`(学習率), `num_round`, `subsample`, `min_child_weight`
- MLA出題：「特徴量の寄与度を説明したい」「テーブルデータで高精度」→ XGBoostが正解になりやすい

**出力と解釈性の整理**
- **出力本体**：分類タスクでは**確率**（または所属クラス）、回帰タスクでは**連続値**
- **Feature Importance**：出力とは別に得られる**付加情報**（モデルから抽出する解釈情報）
- ⚠️ XGBoostのFeature Importanceは**間接的**：寄与の大きさはわかるが**方向（正負）が直接わからない**
  - 「賃金が高いほど予測が上がる」「下がる」かまでは Feature Importance だけでは判定不可
  - 方向まで知るには **SHAP値** を使う（個別予測ごとに正負の寄与を出せる）
- 対比：**線形モデル（ロジスティック回帰・Linear Learner等）は重み係数が直接解釈可能**
  - 係数の符号がそのまま「正なら押し上げ、負なら押し下げ」を示す → XGBoostより解釈が直感的

**MLA出題：「特徴量の方向まで含めて解釈したい」→ 線形モデル or SHAP併用が選択肢**

### RCF（Random Cut Forest）
**教師なし学習**の異常検知アルゴリズム。SageMaker組み込み。
- データをランダムに切断する木（Random Cut Tree）を多数作る
- 孤立しやすい（少ない切断で分離される）点＝**異常スコアが高い**
- 各データポイントに**異常スコア**を付与（高いほど異常）
- 用途：**時系列の外れ値・不正検知・センサー異常・ストリーミングデータ監視**
- Kinesis Data Analytics でもRCFが使える（リアルタイム異常検知）
- ラベル不要なので、異常データが手元にない場合に最適
- MLA出題：「ラベルなし」「異常検知」「時系列の外れ値」→ RCFが正解パターン

### ナイーブベイズ分類器（Naive Bayes Classifier）
**ベイズの定理**に基づく確率的な教師あり分類アルゴリズム。
「特徴量同士は互いに独立」と仮定（=ナイーブ／単純）して計算を超軽量化。

**仕組み**
- ベイズの定理：`P(クラス|特徴) ∝ P(クラス) × P(特徴|クラス)`
- 各特徴の条件付き確率を**掛け合わせる**だけでクラス確率を算出
- 最も高い確率のクラスを予測結果とする

**主なバリエーション**
- **多項分布（Multinomial）NB**：単語の出現回数を扱う → テキスト分類で定番
- **ベルヌーイNB**：単語の有無（0/1）を扱う
- **ガウスNB**：連続値（特徴量が正規分布前提）を扱う

**メリット**
- **超高速・少ないデータでも動く**（学習・推論ともに軽い）
- 高次元スパースデータ（テキスト）に強い
- 実装がシンプルでベースラインに最適
- TF-IDF / Bag of Words と相性◎

**デメリット**
- 「特徴量が独立」という仮定が強い（実際には相関あり）→ 精度に上限
- 学習時に未出現の単語は確率0になる → **ラプラス平滑化（Laplace Smoothing）** で対処
- 特徴量間の相互作用は捉えられない

**使われるケース**
- **スパムメール判定**（古典的鉄板用途）
- **テキスト分類**（ニュースカテゴリ・感情分析）
- **文書のトピック分類**
- ベースラインモデル（まずNBで精度を測ってから複雑なモデルへ）

**MLA出題ポイント**
- 「軽量・高速にテキスト分類」「スパム判定」→ **ナイーブベイズ**
- 「TF-IDF特徴量で分類」→ ナイーブベイズ or 線形SVMが定番
- 「特徴量の独立性を仮定するアルゴリズム」→ ナイーブベイズ

### SVM（Support Vector Machine）
分類と回帰の両方に使用できる教師あり学習モデル。
データを最大マージンで分離する超平面を見つける。
- Kernel（カーネル）：RBF等で非線形分離も対応
- ヒンジロス = SVMの損失関数
- SageMakerでは Linear LearnerでSVM（線形）の実装が可能

### ロジスティック回帰
二値分類で確率（0〜1）を出力する手法。
シグモイド関数で線形出力を0〜1に変換。
- 「更新・解約」「スパムかどうか」等の二値予測に使用
- 閾値デフォルト0.5で分類（ビジネス要件で変更可能）

### シグモイド関数
どんな実数値でも0〜1に変換するS字カーブ関数。
ロジスティック回帰の出力層で使用。

### Factorization Machines（因数分解機）
高次元スパースデータで特徴間の相互作用を効率よく学習するアルゴリズム。
- 用途：クリック予測、レコメンドシステム
- データ形式：RecordIO-Protobuf推奨（CSV非推奨）
- 特徴：ペアワイズ（2次）の特徴間相互作用をキャプチャ

### Linear Learner
SageMaker組み込みアルゴリズム。**二値分類・多クラス分類・回帰**の3つに対応する線形モデル。
- 分類時：内部的に**ロジスティック回帰として動作**（シグモイドで確率出力）
- 回帰時：線形回帰として動作
- **重み係数が直接解釈可能**（線形モデル全般の特徴）
  - 係数の符号 = 予測への押し上げ/押し下げ方向
  - XGBoostのFeature Importanceより解釈性が高い
- 並列学習・自動チューニング・複数損失関数の同時試行に対応
- SVM（線形）の実装にも使える
- MLA出題：「線形で解釈性が高い分類/回帰」「重み係数を解釈したい」→ Linear Learner

### ニューラルネットワーク（NN）／ 多層パーセプトロン（MLP）
複数の層（入力層・隠れ層・出力層）から成る学習モデル。
- **ニューラルネットワーク = 多層パーセプトロン（MLP）**：同じものを別名で呼んでいるだけ
- 各層は重み・バイアス・活性化関数で構成
- 学習：誤差逆伝播法（バックプロパゲーション）+ 勾配降下法
- 用途：非線形な関係の学習全般（テーブル・画像・テキスト）
- 拡張：CNN（画像向け）、RNN/LSTM（系列向け）、Transformer（注意機構ベース）

### Object2Vec
SageMaker組み込みアルゴリズム。**任意オブジェクトを固定長ベクトルに埋め込む**汎用Embeddingアルゴリズム。
- 入力：ペア（オブジェクトA, オブジェクトB, 関係ラベル）
- 出力：オブジェクトごとの埋め込みベクトル
- 用途：
  - **類似アイテム検索・レコメンド**（ユーザ-アイテムの関係を学習）
  - 文-文の類似度、文書クラスタリング
  - グラフのノード埋め込み
- 似た役割のWord2Vecよりも**自由なペア関係**を扱える
- MLA出題：「ユーザとアイテムの関係をベクトル化してレコメンド」「任意ペアの埋め込み」→ Object2Vec

### LDA（Latent Dirichlet Allocation / 潜在ディリクレ配分）
**確率的トピックモデル**。文書がどのようなトピックの混合で成り立っているかを推定する教師なし学習。
- 各文書はトピック分布を持ち、各トピックは単語分布を持つ
- 入力：単語の出現回数（Bag of Words）
- 出力：文書ごとのトピック割合、トピックごとの代表単語
- 計算量がNTMより重い → **中規模データ向け**
- SageMaker組み込みアルゴリズムあり
- 似たサービス比較：
  - **LDA**：確率モデル、解釈性が高い、中規模向き
  - **NTM（Neural Topic Model）**：ニューラルベース、大規模・高速
- MLA出題：「小〜中規模文書のトピック抽出」「確率的トピックモデル」→ LDA

### 協調フィルタリング
ユーザーの過去の行動・評価に基づいて推薦を行う手法。
「あなたと似た好みの他ユーザーが好きなアイテムをあなたに勧める」。
- ユーザーベース：似た好みのユーザーから推薦
- アイテムベース：アイテム間の類似度で推薦
- 弱点：コールドスタート問題（新商品・新ユーザーに弱い）

### CNN（畳み込みニューラルネットワーク）
画像データの特徴を効率よく抽出するニューラルネットワーク。
コンピュータビジョン全般に使用。
- 画像分類、物体検出、セグメンテーション、顔認識
- 構成：畳み込み層 → プーリング層 → 全結合層

### セマンティックセグメンテーション（Semantic Segmentation）
画像を**ピクセル単位でクラス分類**する手法。
- 出力：入力画像と同じサイズのマスク（各ピクセルにクラスラベル）
- 同じクラスの物体は区別しない（人A・人Bは同じ「人」ラベル）
- 用途：自動運転（道路/車/歩行者の領域認識）、医療画像（腫瘍領域抽出）、衛星画像解析
- 代表モデル：U-Net、FCN、DeepLab
- SageMaker組み込みアルゴリズム「Semantic Segmentation」あり
- MLA出題：「ピクセル単位でクラス判定」「医療画像で病変領域を塗り分け」→ セマンティックセグメンテーション

### オブジェクト検出（Object Detection）
画像内の**物体の位置（バウンディングボックス）とクラス**を同時に出力する手法。
- 出力：[x, y, 幅, 高さ, クラス, 信頼度] のリスト
- セマンティックセグメンテーションと違い**個体を区別**（人A・人Bを別オブジェクトとして検出）
- 用途：自動運転（前方の車を個別検出）、監視カメラ、製造業の検品
- 代表モデル：YOLO、SSD、Faster R-CNN
- SageMaker組み込みアルゴリズム「Object Detection」あり
- MLA出題：「画像から物体の位置とラベルを返す」→ オブジェクト検出

**画像系タスクの違いまとめ**
| タスク | 出力 | 個体区別 |
|---|---|---|
| 画像分類 | 1枚に1ラベル | - |
| オブジェクト検出 | 矩形+ラベル（複数） | ✅ |
| セマンティックセグメンテーション | ピクセル毎ラベル | ❌（同クラス統合） |
| インスタンスセグメンテーション | ピクセル毎ラベル+個体ID | ✅ |

## 推論方式

### リアルタイム推論
モデルがメモリに常駐した状態で、リクエストごとに即時応答。
- 制約：60秒・5MB程度
- 低レイテンシーが可能（モデルが常駐しているため）
- コスト：高い（常時課金）

### バッチ推論
大量データをまとめて処理。リアルタイム性は不要、定期実行向け。
コスト：低い（実行時のみ）

### 非同期推論
処理は後で実行し、結果を後から取得。
重い処理・長時間処理に適している。

### サーバーレス推論
コールドスタートあり。トラフィックが断続的でコストを抑えたい場合に有効。

## 評価指標

### 回帰系

| 指標 | 計算方法 | 外れ値への感度 |
|---|---|---|
| MAE（平均絶対誤差） | \|予測-実測\|の平均 | 低い（等しく扱う） |
| MSE（平均二乗誤差） | (予測-実測)²の平均 | 高い（二乗でペナルティ） |
| RMSE（二乗平均平方根誤差） | MSEの平方根 | 高い（MSEと同じ） |

- 外れ値が多いデータ → **MAE**（ロバスト）
- 大きな誤差を特に避けたい → **MSE / RMSE**

### 分類系

#### Log Loss
確率予測の誤差を測る指標。
自信を持って外した場合に大きくペナルティを与える。

#### AUC-ROC（Area Under the ROC Curve）
分類モデルの判別性能を評価する指標。閾値に依存しない。
- ROC曲線：X軸=偽陽性率、Y軸=真陽性率
- AUC値：0.5（ランダム）〜1.0（完璧）
- 用途：モデル自体の評価（閾値決定は別の話）
- 不均衡データでも有効

#### Perplexity（パープレキシティ）
言語モデルが「次の単語の予測にどれだけ迷っているか」を数値化。
- 低い → 次の単語を高い確信で予測できる（よく学習されている）
- 高い → 迷っている（学習が不十分）
- 用途：LLMの性能比較

#### PSNR（Peak Signal-to-Noise Ratio / ピーク信号対雑音比）
**画像・動画・音声の品質**を、元データと再構成データの差で評価する指標。
- 計算：`PSNR = 10 × log10(MAX² / MSE)`（単位：dB）
  - MAX：信号の最大値（画像なら255）、MSE：元と再構成の平均二乗誤差
- 値が**高いほど品質が良い**（ノイズが少ない）
  - 30dB未満：劣化が目立つ
  - 30〜40dB：許容範囲
  - 40dB以上：高品質（人間の目では区別困難）
- 用途：**画像圧縮・超解像・ノイズ除去・GAN画像生成**の評価
- 欠点：人間の知覚と必ずしも一致しない（補助でSSIMを併用することも）

#### ARI（Adjusted Rand Index / 調整ランド指数）
**クラスタリング結果**を正解ラベルと比較して評価する指標（外部評価指標）。
- 同じクラスタに分類されたペアと、異なるクラスタに分けられたペアの一致率を測り、偶然の一致を補正したもの
- 値の範囲：**-1 〜 +1**
  - **1.0**：完全一致（クラスタリング結果と正解が完全一致）
  - **0**：ランダム分割と同等（無意味）
  - **マイナス**：ランダムより悪い
- 用途：**K-means等のクラスタリング評価**、正解ラベルがある場合のクラスタ精度測定
- 通常のRand Indexと違い、クラスタ数やサイズの影響を補正している点が強み
- MLA出題：教師なし学習の評価指標選択肢として登場（PSNRと混同しないこと）

### 公平性指標
- **DI（Disparate Impact）**：2つのグループが異なる結果を受ける程度を測定
- **Demographic Parity**：異なるグループ間で肯定的結果の確率が同じかを評価
- SageMaker Clarifyで自動計算可能

#### CI（Class Imbalance / クラス不均衡）
**保護属性グループ（ファセット）間のサンプル数の偏り**を測る指標。
- 計算：`(他グループの数 - 注目グループの数) / 全体`
- 範囲：-1〜+1（0が均衡、絶対値が大きいほど偏り）
- **見るのは「グループの人数」だけ**（ラベルは見ない）
- 例：男性8000人 vs 女性2000人 → CI = (8000-2000)/10000 = 0.6

#### DPL（Difference in Proportions of Labels / ラベルの比率の差）
**グループごとの「正例（ポジティブラベル）の割合」の差**を測る指標。
- 計算：`P(Y=1|グループA) - P(Y=1|グループB)`
- 範囲：-1〜+1（0が公平）
- **ラベル分布の偏り**を見る（学習前のデータバイアス検出）
- 例：男性の採用率50%、女性の採用率20% → DPL = 0.30

#### CI と DPL の違い
| | CI | DPL |
|---|---|---|
| 見るもの | グループ間の**人数の比率** | グループ間の**正例ラベル比率** |
| 目的 | データ量の偏り検出 | ラベル付与の偏り（差別の兆候）検出 |
| ラベル | 不要 | 必要 |
| 例 | 男8000:女2000 | 男採用率50%:女採用率20% |

→ **CIは「サンプル数」、DPLは「ラベルの当たり確率」の差**。
両方とも SageMaker Clarify の**学習前バイアス指標（Pre-training Bias）**として自動計算される。

#### 学習前バイアス指標 評価の目安
| 指標 | OK（公平） | NG（偏りあり） |
|---|---|---|
| CI | 0付近（例：±0.1以内） | 絶対値0.5以上：データ量が大きく偏る |
| DPL | 0付近 | 絶対値0.1超：ラベル付与に明確な差。0.3以上は危険 |

例（NG）：男性合格率50% / 女性合格率20% → DPL=0.30 → このまま学習させない、再サンプリング・公平性制約等で対処。

---

#### Post-training Bias（学習後／推論結果のバイアス指標）
モデルが出した**予測ラベル `Ŷ`** を使う指標。Clarifyが推論データ（バリデーション/本番）に対して計算。

##### DPPL（Difference in Positive Proportions in Predicted Labels）
グループ間で**モデルが正例と予測した割合の差**。DPLの予測版。
- 計算：`P(Ŷ=1|A) - P(Ŷ=1|B)`
- OK：0付近（例：±0.05）→ 両グループに同程度の割合で正例予測
- NG：絶対値0.1超 → モデルが片方のグループにだけ正例を多く出している

##### DI（Disparate Impact）
予測正例率の**比**（差ではなく比）。
- 計算：`P(Ŷ=1|不利グループ) / P(Ŷ=1|有利グループ)`
- OK：1.0付近（**0.8〜1.25**が一般的な許容範囲、いわゆる "4/5 ルール"）
- NG：0.8未満 or 1.25超 → 一方のグループが不利な扱い
- 例：男性正例率60%、女性正例率24% → DI=0.4 → NG（女性が著しく不利）

##### AD（Accuracy Difference）
グループ間の**精度の差**。
- 計算：`Accuracy(A) - Accuracy(B)`
- OK：0付近 → どのグループにも同程度の精度
- NG：絶対値0.1超 → 一方のグループだけ予測精度が低い（信頼性に偏り）

##### RD（Recall Difference）
グループ間の**再現率（Recall）の差**。
- 計算：`Recall(A) - Recall(B)`
- OK：0付近
- NG：絶対値0.1超 → 一方の正例を見逃しがち（例：女性候補者の合格者を取りこぼす）

##### DAR / DRR（Difference in Acceptance/Rejection Rates）
- **DAR**：「正例予測のうち本当に正例だった率（Precision相当）」のグループ間差
- **DRR**：「負例予測のうち本当に負例だった率」のグループ間差
- OK：0付近 / NG：絶対値0.1超

#### Post-training 指標 まとめ表
| 指標 | 何を見る | OK目安 | NG目安 |
|---|---|---|---|
| DPPL | 予測正例率の差 | 0付近 | 絶対値>0.1 |
| DI | 予測正例率の比 | 0.8〜1.25 | <0.8 or >1.25 |
| AD | 精度の差 | 0付近 | 絶対値>0.1 |
| RD | Recallの差 | 0付近 | 絶対値>0.1 |
| DAR | Precisionの差 | 0付近 | 絶対値>0.1 |

→ **Pre-training（CI/DPL）でデータの偏り** → **Post-training（DPPL/DI/AD/RD）でモデル出力の偏り**を確認、の二段構え。

## 検証手法

### k-fold交差検証
データをk個に分割して、k回の学習・評価を繰り返す手法。
- 全データをテストに使える（データ効率が良い）
- 結果がより信頼できる
- 小規模データセット向け

## データ品質・信頼性

### データリーケージ
モデルが「本来知るべきでない情報」を訓練時に使ってしまうこと。
例：テストデータのmin/maxで訓練データを正規化 → テスト情報が漏れる

### Train-Serving Skew（トレーニング・サービングスキュー）
訓練時と推論（サービング）時でデータ処理が異なることによる性能劣化。
例：訓練時min=0/max=100で正規化 → 推論時min=0/max=200（基準が違う）
- 対策：SageMaker PipelinesやFeature Storeで前処理を共通化

### レコードリンケージ
異なるデータソースから同一人物・同一オブジェクトを特定して統合する処理。
名前・住所などの類似度から確率的に同一か判定する。

## 模試ミス回収メモ

### EDA（探索的データ分析）
モデル構築前にデータの傾向・分布・欠損・外れ値・相関を確認する分析工程。
- 確認内容：基本統計量、欠損値、外れ値、目的変数と説明変数の関係、特徴量相関、クラス不均衡
- 目的：データ理解、前処理方針決定、特徴量設計、品質問題発見

**判定キーワード**
- 「モデル学習前にデータの性質を調べる」「分布・欠損・相関の確認」→ **EDA**

### 特徴エンジニアリング（Feature Engineering）
生データから機械学習モデルに適した特徴量を選択・変換・作成するプロセス。
- 不要特徴量削除、欠損値補完、エンコーディング、標準化、日付分解、特徴量組み合わせ、外れ値処理、ビニング

**判定キーワード**
- 「生データをモデルに効く形へ加工」「特徴量の選択・変換・作成」→ **Feature Engineering**

### ハイパーパラメータ vs モデルパラメータ
| | ハイパーパラメータ | モデルパラメータ |
|---|---|---|
| 設定タイミング | 学習前に人が決める | 学習中にデータから獲得 |
| 例 | 学習率、エポック数、バッチサイズ、層数、決定木深さ、正則化係数、k-meansのk | NN重み・バイアス、線形回帰係数、ロジ回帰係数 |
| チューニング対象 | ✅ | ❌（学習で自動更新） |

**判定キーワード**
- 「チューニング対象」「学習前に人が決める」→ ハイパーパラメータ
- 「学習で更新される値」→ モデルパラメータ

### トレーニング / バリデーション / テストセット
- **Train**：学習に使う
- **Validation**：ハイパラ調整・モデル選択・early stopping。必須ではない（交差検証で代替可）
- **Test**：最終評価・汎化性能確認。**ハイパラ調整に使うとリーク**

**判定キーワード**
- 「ハイパラ調整・モデル選択」→ Validation
- 「最終評価・汎化性能」→ Test
- 「テストセットで調整」→ 誤答パターン

### AdaBoost / Gradient Boosting / XGBoost
| | 特徴 |
|---|---|
| **AdaBoost** | 誤分類データに重みを上げて次の弱分類器で重点学習。外れ値・ノイズに弱い |
| **Gradient Boosting** | 前モデルの誤差を勾配で補正。柔軟だが調整必要 |
| **XGBoost** | GB拡張版。正則化・欠損値処理・`scale_pos_weight`・並列化。不均衡データに強い |

**判定キーワード**
- 「誤分類データの重みを上げる」→ AdaBoost
- 「勾配で誤差補正」→ Gradient Boosting
- 「不均衡データ・不正検知・表形式で高精度」→ XGBoost

### アンサンブル学習
| 手法 | モデル | 学習 | 目的 | 代表例 |
|---|---|---|---|---|
| **Bagging** | 同種が多い | 並列 | 分散を下げる | Random Forest |
| **Boosting** | 同種が多い | 逐次 | 誤差補正で精度向上 | XGBoost, AdaBoost |
| **Stacking** | 異種も可 | ベース＋メタ | 複数モデルの強みを統合 | メタ=RF/XGB |
| **Voting** | 異種も可 | 単純統合 | 多数決・平均 | Voting Classifier |

**判定キーワード**
- 「異なるモデルを組み合わせ・メタモデルで統合」→ Stacking
- 「同じ弱学習器を順番に強くする」→ Boosting
- 「並列に同種モデル→平均」→ Bagging
- 「単純多数決」→ Voting

### レコメンデーションのハイブリッドモデル
- 複数モデル（協調FL・コンテンツベース・DL）が**顧客セグメント別に強み**を持つ場合
- 各モデルの予測を**重み付き平均で組み合わせ**
- 問題文に「顧客セグメントごとの得意不得意」が明示されたら**重み付きハイブリッド**を優先（Stackingではない）

### アソシエーションルール学習
データ内の項目同士の同時発生パターンを発見する**教師なし学習**。
- 代表アルゴリズム：**Apriori**
- 用途：マーケットバスケット分析、購買分析、商品組み合わせ

**判定キーワード**
- 「一緒に買われやすい組み合わせ」「Apriori」「マーケットバスケット」→ アソシエーションルール学習

### 教師あり学習とニューラルネットワーク
- NNは**教師あり学習でも使われる**（分類・回帰）
- NNは「学習方式」ではなく「モデル構造」
- 教師なし・自己教師あり・強化学習でもNNは使う

### 半教師あり学習（Semi-supervised Learning）
**少量のラベル付き + 大量のラベルなし**を組み合わせる学習。
- ラベル付けコストが高い場面で有効
- 文書分類で「一部だけラベルあり」→ 半教師あり学習
- 全てラベルありなら教師あり学習

### TensorFlow Estimator API
TF 1系の高レベルAPI。学習・評価・推論・保存・分散学習を統一的に扱う。
- TF 2系では `tf.keras` が中心、Estimatorは移行対象
- 「train / evaluate / predict / 分散学習をまとめる」→ Estimator API

### カタストロフィック・フォーゲッティング
新しいタスク・データで追加学習した結果、**以前学習した知識・性能を大きく失う**現象。継続学習・転移学習・FTで発生。

**対策**
- **リプレイ / リハーサル**：古いタスクのデータも少量混ぜる
- **知識蒸留**：旧モデルの出力を教師に
- **正則化（EWC等）**：重要な重みを大きく変えない
- **パラメータ分離**：ベースモデル凍結 + Adapter / LoRA / タスク別ヘッド

**判定キーワード**
- 「新しいタスク学習で古い性能が落ちる」→ カタストロフィック・フォーゲッティング
- 対策候補：旧データ混合、知識蒸留、EWC、凍結、Adapter、LoRA

### 転移学習
学習済みモデルを別タスクに再利用する手法。
- 忘却対策には、ベースモデルの大部分を**凍結**し、最後の層だけ再学習
- 学習率を小さくする、旧データ・知識蒸留を併用

### プルーニング（Pruning）
モデルの不要な部分を削除して**軽量化・高速化・過学習抑制**。
- **決定木**：重要度の低い枝を削る → 過学習抑制
- **ニューラルネット**：重要度の低い重み・ニューロン・チャネルを削除 → モデルサイズ・推論コスト削減
- 用途：エッジデバイス最適化、レイテンシ削減

### AUC-ROC 補足
- ROC曲線が**左上に近いほど高性能**
- AUC=1：完璧 / AUC=0.5：ランダム相当 / **AUC≈0：逆方向に学習している可能性**

### Shapley値 vs PDP
| | Shapley値 | PDP（Partial Dependence Plot） |
|---|---|---|
| 粒度 | **個別インスタンス**の予測寄与 | **データ全体**で特徴量の平均的影響 |
| 説明範囲 | ローカル説明 | グローバル説明 |

**判定キーワード**
- 「個別予測の寄与」→ Shapley値
- 「特徴量を変えると平均予測がどう動くか」→ PDP

### CDD（Conditional Demographic Disparity）
公平性指標の一種。**条件付き**でグループ間の予測差を評価。
- 単純な人口統計差ではなく、条件を揃えた上での差を見る
- バイアス検出・公平性評価の文脈
